# forestworld-web

Frontend for ForestWorld — Amazon forest-risk intelligence.

Backend + data pipeline live in [`climate_risk_track`](https://github.com/angiesanchezm/climate_risk_track/tree/main). That pipeline writes the PNGs you see in `web_assets/` and produces the `aois.json` this app consumes.

## Run locally

```
python -m http.server 8000
```

Open http://localhost:8000.


## Project layout

```
forestworld-web/
├── index.html        ← styles, CDN script tags
├── app.jsx           ← router (selector ↔ case page)
├── data.jsx          ← AOI catalog, path helpers, MapBiomas legend
├── views.jsx         ← TopBar, LocatorMap, SelectorView, CasePage
├── raster.jsx        ← viewer primitives (ImageStage, period picker)
├── vercel.json       ← static-site config
└── web_assets/       ← rasters + manifest (regenerated by `climate_risk_track`)
    ├── manifest.json
    ├── maps/peru_locator.png
    └── <aoi_id>/
        ├── metadata.json
        ├── sentinel_<YYYY>.png         (7 years: 2018–2024)
        ├── mapbiomas_<YYYY>.png        (5 years: 2018–2022)
        └── damage_<from>_<to>.png      (4 transitions, with _overlay + _thumb variants)
```

The 4 AOIs are `boca_manu`, `la_pampa`, `masisea`, `chipiar`.

## Regenerate web_assets

The PNGs and `aois.json` are produced by the data pipeline in [`climate_risk_track`](https://github.com/angiesanchezm/climate_risk_track/tree/main). Clone that repo as a sibling on disk and run its pipeline.

## Deploy to Vercel


```
npm i -g vercel
cd forestworld-web
vercel
```

It asks 3 questions, gives you a `*.vercel.app` URL with HTTPS. Subsequent `vercel --prod` redeploys.

Alternative: push this repo to GitHub, then import it on vercel.com → it auto-deploys on every push. Or drag the folder onto vercel.com.

No build step needed — Vercel serves the directory as static files.
