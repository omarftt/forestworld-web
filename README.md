# forestworld-web

Static frontend for ForestWorld — a 4-AOI Amazon forest-risk demo. React + Babel-standalone, no build step. Reads PNG rasters + JSON metadata from `web_assets/`.

Sibling repo: [`forestworld-core`](../forestworld-core) (data pipeline + backend). The pipeline writes generated PNGs straight into this repo's `web_assets/`.

## Run locally

```
python -m http.server 8000
```

Open http://localhost:8000.

A static server is required — opening `index.html` via `file://` will fail because the JSX files are loaded with `<script src>`.

## Project layout

```
forestworld-web/
├── index.html        ← styles, CDN script tags
├── app.jsx           ← router (selector ↔ case page)
├── data.jsx          ← AOI catalog, path helpers, MapBiomas legend
├── views.jsx         ← TopBar, LocatorMap, SelectorView, CasePage
├── raster.jsx        ← viewer primitives (ImageStage, period picker)
├── vercel.json       ← static-site config
└── web_assets/       ← rasters + manifest (regenerated by forestworld-core)
    ├── manifest.json
    ├── maps/peru_locator.png
    └── <aoi_id>/
        ├── metadata.json
        ├── sentinel_<YYYY>.png         (7 years: 2018–2024)
        ├── mapbiomas_<YYYY>.png        (5 years: 2018–2022)
        └── damage_<from>_<to>.png      (4 transitions, with _overlay + _thumb variants)
```

The 4 AOIs are `boca_manu`, `la_pampa`, `masisea`, `chipiar`.

> **Optional · how is this data generated?** If you want to understand the preprocessing that produces the rasters, damage masks and ranked cases, see [`preprocessing/README.md`](preprocessing/README.md). It is a read-only mirror of the pipeline that lives in the sibling `climate_risk_track` repo, included here as a reference.

## Regenerate web_assets

From the sibling repo:

```
cd ../forestworld-core
source venv/bin/activate
cd data_prep && python web_asset.py
```

It writes back into this repo's `web_assets/` via the relative path `../../forestworld-web/web_assets/`. Both repos must be siblings on disk for this to work.

## Deploy to Vercel

Easiest path (no Git required):

```
npm i -g vercel
cd forestworld-web
vercel
```

It asks 3 questions, gives you a `*.vercel.app` URL with HTTPS. Subsequent `vercel --prod` redeploys.

Alternative: push this repo to GitHub, then import it on vercel.com → it auto-deploys on every push. Or drag the folder onto vercel.com.

No build step needed — Vercel serves the directory as static files.
